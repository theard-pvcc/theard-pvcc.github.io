Chapter 2: Algorithms

  1. Explain why natural language is not an effective tool for describing algorithms.
    • Natural language is not an effective tool for algorithms because it is too verbose, making it difficult for the reader to find specific sections.
  2. Problem 1:
    • A student has three grades; the final grade is the average of the three. List the algorithmic steps to find the final grade.
      1. Get the values for grade 1, grade 2, grade 3
      2. Set the value of final grade to ((grade 1 + grade 2 + grade 3) / 3)
      3. Print the value of final grade
      4. Stop
  3. Problem 2:
    • A car owner recorded the car's previous car mileage, and the number of gallons of gas the owner filled their tank with. List the algorithmic steps to find the car's mpg.
      1. Get the values for starting mileage, ending mileage, gallons filled, ending gallons
      2. Set the value of distance driven (ending mileage - starting mileage)
      3. Set the value of gallons used to (gallons filled - ending gallons)
      4. Set the value of mpg to (distance driven + gallons used)
      5. Print the value of mpg
      6. Stop

Chapter 2: Briefly define:

  1. natural language
    • Natural language is a human spoken/ written language (English, Spanish, French, etc...)
  2. psuedocode
    • English-language constructs designed to resemble statements in a programming language.
  3. sequential operation (straight-line algorithm)
    • An algorithm that takes a series of operations in a specific order of steps. Using some "input" do some "computation" to get some "output"
  4. input
    • Operation that provides data to computing agent from outside world
  5. output
    • Send results from computing agent to outside world
  6. conditional statement
    • Statement that requires some question to be answered before executing some computation.
  7. boolean condition
    • A condition that returns either "true" or "false"
  8. algorithmic discovery process
    • The steps taken to find a solution to a given problem

Chapter 3: Briefly describe or define:

  1. why algorithms should be correct
    • It offers no value if the result that the algorithm returns is incorrect. If you have a nice car, but it doesn't start, it has no use.
  2. why algorithms should be efficient
    • Algorithms are intended to be used to solve real world problems. Therefore, an inefficient algorithm may waste time or storage space.
  3. program maintenance
    • Fixing errors uncovered through use, and keeping up with developing requirements.
  4. importance of algorithm: ease of understanding
    • The algorithm being used should be easy to understand to make maintenance as easy as possible.
  5. algorithmic elegance
    • The algorithmic equivalent to style.
  6. sequential search algorithm
    • Algorithm used to step through a list or directory to find a specific element.
  7. order of magnitude
    • The number of times an algorithm must iterate to achieve the correct answer.
  8. selection sort
    • Algorithm used to sort a list of elements from back to front
  9. flop (floating point operation)
    • Used to measure the performance of a computer.
  10. teraflop
    • 1 teraflop = 1 trillion flops
  11. petaflop
    • 1 petaflop = 1 quadrillion flops
  12. parallel processor
    • utilizing 2 CPUs to handle seperate parts of an overall task
  13. data cleanup problem
    • Dealing with input data that is not valid to the data we are attempting to collect
  14. binary search algorithm
    • Used to search a sorted list from the middle rather than the start or end
  15. pattern matching
    • Finds all occurences of a pattern.
