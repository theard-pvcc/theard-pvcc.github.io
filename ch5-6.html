Computer Architecture Terms

  1. RAM
    • Random Access Memory
    • Memory is divided into fixed-size units called cells which are each associated with an address
    • All accesses to memory are to a specified address
    • The time it takes to fetch or store the contents of a single cell is the same for all cells in memory
  2. cache
    • Smaller high speed memory unit between processor and main memory
  3. Principle of locality: The high likelihood that when a program fetches a piece of data or an instruction:
    1. It will access that same instruction or piece of data in the very near future.
    2. It will soon access the instructions or data that are located near that piece of data, where "near" means an address whose numerical value is close to this one.
  4. ROM
    • Read-only memory- information has been prerecorded during manufacture, and can't be modified or removed
  5. memory cell size
    • Number of bits per cell
  6. memory cell address
    • Unique identifier of a cell. Unsigned integers 0, 1, 2, ..., MAX
  7. maximum memory size
    • The value 2^N
  8. register
    • Special high-speed storage cell that holds the operands of an arithmetic operation and its results.
  9. fetch
    • Get a copy of the contents of the memory cell with a specified address and return those contents as the result of the operation.
  10. store
    • Store the specified value into the memory cell specified by the address.
  11. memory access time
    • Time to carry out either a fetch or a store operation
  12. DASD
    • Direct access storage devices
  13. Arithmetic and Logic Unit (ALU)
    • Subsytem that performs math and logic operations. Addition, subtraction, and equality comparisons.
  14. Control Unit
    1. Fetch from memory the next instruction to be executed
    2. Decode it- determine what is to be done
    3. Execute it by issuing appropriate command to the ALU, memory, or I/O controllers

Chapter 6:

Four things the interface between the computer and the hardware does:

  1. Hides the messy details of the underlying hardware from the user
  2. Presents info about what is happening in a way that does not require in-depth knowledge of the internal structure of the system.
  3. Allows easy user access to the resources available on this computer.
  4. Prevents accidental or intentional damage to hardware, programs, and data.

Responsibilites of systems software:

  1. Hides the complex and unimportant details of internal structure
  2. Presents important info to the user in a way that is easy to understand
  3. Allows the user to access machine resources in a simple and efficient way
  4. Providese a secure and safe environment in which to operate

Terms:

  1. Systems Software
    • A collection of computer programs that manage the resources of a computer and facilitates access to those resources.
  2. User interface
    • A graphical user interface that gives users a visual overview and graphical control of the computers services and capabilities.
  3. language services
    • Programs called assemblers, compilers and interpreters that allo the user to write programs in high-level languages.
  4. memory managers
    • Programs that allocate mempry space for programs and data and retrieve this memory space when it is no longer needed.
  5. information managers
    • Handle the organization, storage and retrieval of info on storage devices.
  6. i/o managers
    • Software packages that allow you to easily and efficiently use the different types of input and output devices that exist on modern computer systems.
  7. scheduler
    • System program that keeps a list of programs ready to run on the processor. Selects the on that will execute next.
  8. utilities
    • Collections of library routines that provide a wide range of useful services to a user or to other system routines.
  9. assembly language
    • Language designed for both computers and people to understand.

Functions of an Operating System

  1. User interface management
  2. Control of access to the system and to data files
  3. Program scheduling and activation
  4. Efficient resource allocation
  5. Deadlock detection and error detection
