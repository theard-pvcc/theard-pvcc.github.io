  • List four disadvantages of using assembly language:
    1. Programmer must manually manage the movement of data items
    2. Programmer must take a microscopic view of a task
    3. machine specific
    4. statements are not natural-lang-like
  • List the four expectations programmers have for writing programs in a high-level language:
    1. Programmer need not manage the details of the movement of data items
    2. Programmer can take a macroscopic view of tasks
    3. programs are portable
    4. Programming statements are closer to natural language
  • Briefly describe or define:
    1. Object code: binary machine language instructions
    2. Source code: high-level language instructions
    3. procedural language: imperative languages- consist of sequences of statements that manipulate data
    4. Integrated Development Environment (IDE): development environment used to assist in the completion of a software project
    5. Software development life cycle: overall sequence of steps needed to complete a large-scale software project
    6. waterfall model of software development: model for progressively moving development down a series of plateaus
    7. agile model of software development: flexible development cycle that admits that problem specs are never a done deal.
  • Describe each of these components of the software development life cycle:
    1. feasibility study: evaluates a proposed project and compares the costs and benefits of various solutions.
    2. Problem specification: developing a clear, concise and unambiguous statement of the exact problem
    3. program design: time to plan how it is to be done
    4. algorithm selection: Once subtasks have been identified, algorithms must be found to carry them out.
    5. coding: process of converting the detailed designs into computer code
    6. debugging: resolving issues with the code
    7. testing: ensuring program runs efficiently and correctly
    8. verification: used to prove that output satisfies certain conditions
    9. documentation: unsuring code is understandable by others, for future improvements and legibillity
